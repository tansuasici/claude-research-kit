# Reviewer Feedback Index

> The manuscript's compounding memory. Every correction from the author or a
> referee becomes a file in `tasks/reviews/`. Recurring critiques get promoted to
> **Top Rules** below — Claude reads these first at every session start.
>
> This is the analogue of a code kit's "lessons learned." A reviewer's repeated
> complaint ("you keep overclaiming in the discussion") is a *rule*, not a one-off.

## Top Rules

2. Every quantity in the abstract must reappear, identical, in Results.
3. State N, the test, and the effect size + CI for every statistical claim.
